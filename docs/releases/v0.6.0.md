# Release Notes: v0.6.0 — Input UX & Backend Caching

## 🚀 Overview

Welcome to **gitpulse v0.6.0**! This milestone focuses on moving the web application from a "manual input" experience to a polished, searchable, and performance-driven platform. We've introduced server-side caching to significantly reduce summary generation times and a completely new repository discovery system that eliminates the need to manually copy-paste repository names.

---

## ✨ Features

### Searchable Repository Selection
Forget manual text inputs. You can now search your GitHub repositories directly and select multiple projects with a single click.

### High-Performance Caching
We've implemented a robust in-memory caching layer that stores GitHub API responses for up to 10 minutes. This means subsequent summary generations for the same user or repository are now near-instant.

### History Search & Filtering
The history page is now fully searchable. You can filter your past generated summaries by keyword or specify precise date ranges (`start_date` and `end_date`).

### Raw Data Export
Need to include your standup summary in a report or a Jira ticket? Use the new **"Download .md"** button to save a perfectly formatted markdown file directly to your computer.

---

## 🛠️ Technical Hardening

- **Improved Error Clarity**: Instead of failing on a single missing repository, the system now identifies specific issues and generates the rest of your standup summary without interruption.
- **Strict Dependency Pinning**: We've locked all JavaScript and Python dependencies to specific versions to protect against supply-chain attacks and ensure a consistent user experience.
- **Edge Case Tests**: Added a new suite of 23 automated tests covering boundary conditions like empty repository states and malformed date inputs.

---

## 🏗️ Getting Started

To get started with v0.6.0, simply head over to the [GitPulse Dashboard](https://github.com/deepusharma/gitpulse) and log in with your GitHub account.

---

## 🐞 Known Issues & Limitations

- **Private Repositories**: This release is still optimized for public repositories. Private repository support is slated for the upcoming **v0.7** release.
- **Rate Limits**: While caching helps, GitHub API rate limits still apply. Authenticated users (via OAuth) enjoy significantly higher thresholds.
