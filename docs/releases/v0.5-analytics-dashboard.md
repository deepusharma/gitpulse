# Release Notes — v0.5 Analytics Dashboard

## Overview

We are excited to announce the release of the **Analytics Dashboard** for GitPulse! This milestone (v0.5) bridges the gap between raw standup summaries and long-term developer insights, allowing you to visualize your "Git Pulse" directly from the web interface.

## New Features

### 📊 Interactive Commit Frequency

A brand new line-and-bar chart visualization that maps your commit volume over time. 

- Filter by **7, 14, 30, or 90 day** lookback windows.
- Hover over data points for precise commit counts.

### 🥧 Repository Activity Breakdown

Understand where your focus is going across multiple projects.

- Dynamic pie charts visualizing commit distribution by repository.
- Automatic percentage calculation based on actual GitHub activity.

### 🔥 Productivity Insights Panel

A high-level metric dashboard providing immediate "at-a-glance" stats:

- **Longest Streak**: Track your consecutive days of coding activity.
- **Top Repository**: Identify your most active project in the current window.
- **Most Active Day**: Discover which day of the week you are most productive.
- **Avg. Commits/Day**: Data-driven velocity tracking.
- **Platform Momentum**: Real-time ticker of total summaries generated across GitPulse.

### 🔍 Explore Mode (User/Org Swapper)

The dashboard is no longer restricted to just your own data! 

- **Organization Views**: Enter an organization name (e.g., `google`) to see aggregated activity.
- **Peer Insights**: Enter any GitHub username to visualize their public pulse.
- **Custom Lookback**: A new precision numeric input allows you to specify any window from 1-90 days.

## Experience Polish

- **Seamless Login**: If you are logged in via GitHub, the dashboard now automatically loads *your* data the moment you arrive. 
- **Auto-Fill Generator**: The main standup generator now correctly remembers your username across sessions.
- **Clean Interface**: Fixed navigation redundancy (duplicate headers) and refined numeric input behavior.

## Technical Improvements

- **Direct GitHub Aggregation**: Analytics are generated on-the-fly by querying the GitHub API.
- **PostgreSQL Integration**: Fuses local summary history with live git metrics.
- **NextAuth Session Mapping**: Efficiently maps GitHub OAuth profiles to internal GitPulse identities for faster auto-fill.
- **FastAPI Analytics API**: High-performance backend endpoints serving structured visualization data.

## How to Access

The new dashboard is accessible directly from the **Dashboard** link in the header or by navigating to the `/analytics` route on your deployed application (e.g., `gitpulse.vercel.app/analytics`).
